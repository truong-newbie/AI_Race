## 1. Tổng quan

Sau retrieval, candidates được xếp hạng bằng retrieval score (BM25 + dense + RRF). Reranking là bước thứ hai để cải thiện thứ tự bằng cách kết hợp thêm các tín hiệu domain-specific.

**Hai giai đoạn:**

- **GIAI ĐOẠN 1 (Rule Reranker)**: Weighted feature scoring — nhanh, không cần training, dễ debug.
- **GIAI ĐOẠN 2 (Cross-Encoder Reranker)**: Biểu diễn mention+candidate pair thành một vector — chính xác hơn nhưng chậm hơn.

## 3. GIAI ĐOẠN 1 — Rule Reranker

### 3.1 Base Class

[`src/linking/base_reranker.py`](src/linking/base_reranker.py) định nghĩa interface chung:

```python
class BaseReranker(ABC):
    @abstractmethod
    def rerank(self, candidates, query, mention=None, top_k=10) -> list[RerankResult]:
        pass

    @abstractmethod
    def name(self) -> str:
        pass

@dataclass
class RerankResult:
    code: str
    rerank_score: float
    features: dict[str, float]
    source: str       # primary signal that determined the score
    rank_before: int
```

### 3.2 ICD-10 Rule Reranker

[`src/linking/rule_reranker.py`](src/linking/rule_reranker.py) — class `ICDRuleReranker`.

**ICD Features:**

| Feature | Weight | Mô tả |
|---|---|---|
| `retrieval_score` | 0.94 | Retrieval score (RRF merged) — chiếm ưu thế |
| `lexical_similarity` | 0.02 | Jaccard word overlap + partial overlap |
| `exact_alias` | 0.02 | Exact match với alias/synonym |
| `parent_child_boost` | 0.01 | Parent-child consistency check |
| `context_include` | 0.01 | Include term matching |
| `specificity_penalty` | -0.01 | Exclude term matching |

**Per-batch normalization:**

```python
# _normalize_retrieval dùng max score trong batch, không hardcoded
def set_retrieval_batch(self, candidates):
    self._retrieval_batch_max = max(c.score for c in candidates)

def _normalize_retrieval(self, score):
    if self._retrieval_batch_max == 0:
        return 1.0 if score > 0 else 0.0
    return min(score / self._retrieval_batch_max, 1.0)
```

**Constraint penalty scaling:**

```python
# Penalty được scale theo retrieval confidence
retrieval_conf = features.get("retrieval_score", 0.0)
penalty_scale = 0.3 if retrieval_conf > 0.9 else 0.8
total -= abs(constraint_penalty) * penalty_scale
```

### 3.3 RxNorm Rule Reranker

Class `RxNormRuleReranker`.

**RxNorm Features:**

| Feature | Weight | Mô tả |
|---|---|---|
| `ingredient` | 0.40 | Exact / canonical / brand expansion match |
| `strength_exact` | 0.20 | Exact strength match (e.g., 1000mg) |
| `strength_close` | 0.10 | Within 15% of mentioned strength |
| `unit_match` | 0.10 | Unit match (mg, ml, g) |
| `dose_form` | 0.10 | Dose form match (tablet, injection) |
| `brand` | 0.05 | Brand name match |
| `retrieval_score` | 0.05 | Retrieval signal (low weight) |

**Strength matching logic:**

```python
if abs(parsed_str - entry.strength_value) < 0.01:
    return (1.0, 0.0)  # exact
ratio = abs(parsed_str - entry.strength_value) / parsed_str
if ratio < 0.15:
    return (0.0, 1.0)  # close (within 15%)
```

### 3.4 Unified Reranker

Class `UnifiedRuleReranker` tự động detect ICD vs RxNorm:

```python
def detect_and_rerank(self, candidates, query, mention=None, top_k=10):
    if hasattr(first, "rxcui"):
        return self.rerank_rxnorm(candidates, query, mention, top_k)
    elif hasattr(first, "code"):
        return self.rerank_icd(candidates, query, mention, top_k)
```

## 4. Ontology Constraints

[`src/linking/ontology_constraints.py`](src/linking/ontology_constraints.py) — class `OntologyValidator`.

### 4.1 ICD-10 Constraints

**`check_icd_child_context`**: Child code mà mention thiếu detail → penalty 0.25.

```python
# Child code (parent_code != code) + mention không có detail signals
# → failed, penalty = 0.25
# Parent codes (parent_code == code) → always pass
```

**`check_icd_exclude_terms`**: Mention chứa exclude term → penalty 0.30.

```python
# Nếu mention chứa bất kỳ exclude term nào
# → failed, penalty = 0.30
```

**`check_icd_include_terms`**: Mention chứa include term → boost.

```python
# Mỗi include term match → +0.05 boost
```

### 4.2 RxNorm Constraints

**`check_rxnorm_strength_mismatch`**: Mismatch về strength vẫn được penalize dù dense score cao.

```python
# penalty = |mention_strength - candidate_strength| * 0.1
# Áp dụng ngay cả khi retrieval score cao
# Dense score không che được strength mismatch
```

**`check_rxnorm_dose_form_conflict`**: Tablet vs injection → penalty.

```python
# mention dạng tablet, candidate injection → penalty 0.25
```

### 4.3 Constraint Scaling

Constraints được scale với retrieval confidence:

```python
penalty_scale = 0.3  if retrieval_conf > 0.9  # High confidence → mild penalty
penalty_scale = 0.8  if retrieval_conf <= 0.9  # Low confidence → more aggressive
```

## 5. GIAI ĐOẠN 2 — Cross-Encoder Reranker

[`src/linking/cross_encoder_reranker.py`](src/linking/cross_encoder_reranker.py) — class `CrossEncoderReranker`.

### 5.1 Supported Models

| Model | Domain | Vietnamese |
|---|---|---|
| `dlicari/DRBERT` | Danish + biomedical BERT | Limited |
| `microsoft/BiomedNLP-PubMedBERT-base-uncased-abstract-fulltext` | PubMed abstracts | No |
| `dmis-lab/biobert-v1.1` | PubMed + PMC | No |
| `cross-encoder/ms-marco-MiniLM-L6` | MS MARCO passage ranking | No |

Lazy-loaded: model chỉ được tải khi thực sự cần.

### 5.2 Scoring Format

```
Query: [CLS] mention [SEP] context [SEP]
Candidate: [CLS] code code_name description [SEP]

Raw score → Normalized: 1/(1 + |raw - 3|/3)
```

### 5.3 Hybrid Scoring

Class `HybridCrossEncoderReranker` blend cross-encoder với retrieval:

```python
hybrid_score = alpha * cross_encoder_score + (1 - alpha) * retrieval_score
```

### 5.4 Training Dataset Builder

[`src/linking/cross_encoder_dataset.py`](src/linking/cross_encoder_dataset.py) — class `CrossEncoderDatasetBuilder`.

**Hard negatives cho ICD:**

```python
# Same chapter, different subcategory
# Same alias, different code
# Parent-child siblings
```

**Hard negatives cho RxNorm:**

```python
# Same ingredient, different strength
# Same brand, different ingredient
# Dose form mismatch
```

**Output format:** SBERT `InputExample` hoặc JSON.

### 5.5 Fine-tuning


### Weight Evolution

```
Initial ICD weights:  retrieval=0.10  → worse=4
v2 (conservative):     retrieval=0.65  → worse=6
v3 (more conservative):retrieval=0.95  → worse=9  ← normalization bug
v4 (fixed norm):        retrieval=0.95  → worse=2  ← constraint penalty bug
v5 (penalty scale):     retrieval=0.94  → worse=0  ← ✓ neutral
```

**Root causes of regressions:**

1. `_normalize_retrieval` used hardcoded divisor (0.01) → all candidates normalized to ~1.0, collapsing rank order.
2. `constraint_penalty` applied directly (unweighted) → large ontology penalties overwhelmed retrieval score.
3. Fix: per-batch max normalization + retrieval-confidence-based penalty scaling.

## 7. Kết luận và khuyến nghị

### Cross-Encoder vs Rule Reranker

| Criteria | Rule | Cross-Encoder |
|---|---|---|
| Speed | Fast (no GPU needed) | Slow (GPU required) |
| Accuracy | Neutral (0 delta) | Expected +1-3% |
| ICD accuracy | Already 100% R@1 | Likely no gain |
| RxNorm accuracy | Neutral | Potential gain |
| Debugging | Easy (feature-level) | Hard (black-box) |
| Training needed | No | Yes (hard negatives) |

### Recommendation

1. **ICD-10**: Giữ Rule Reranker. Retrieval đã đạt 100% R@1 — không cần thay đổi.
2. **RxNorm**: Cross-Encoder có tiềm năng cải thiện 5% miss cases. Nếu cần:
   - Train cross-encoder với hard negatives (same ingredient, different strength)
   - Sử dụng `HybridCrossEncoderReranker` với `alpha=0.3` để blend với retrieval
3. **End-to-end pipeline**: Tích hợp Rule Reranker như một step trung gian, không thay thế retrieval.

## 8. API và cách sử dụng

### Rule Reranker

```python
from src.linking.rule_reranker import ICDRuleReranker, RxNormRuleReranker
from src.linking.icd.hybrid_retriever import HybridRetriever, MergeConfig

# ICD
entries = get_knowledge_base()
retriever = HybridRetriever(entries=entries, merge_config=MergeConfig(...), top_k=20)
reranker = ICDRuleReranker(entries)

candidates = retriever.retrieve(query, mention=mention, top_k=20)
results = reranker.rerank(candidates, query, mention=mention, top_k=10)

for r in results:
    print(f"{r.code}  score={r.rerank_score:.4f}  source={r.source}")
    print(f"  features: {r.features}")
```
