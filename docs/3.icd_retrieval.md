## 1. Tổng quan

Với mỗi entity `CHẨN_ĐOÁN`, trả về danh sách top-k mã ICD-10 có khả năng phù hợp nhất.

**Input**: mention, câu chứa mention, đoạn context, entity assertions.

**Output**: Danh sách candidates với code, score, sources.

```json
[
  {"code": "K21.9", "score": 0.91, "sources": ["exact", "bm25", "dense"]}
]
```

```
    │
    └── HybridRetriever

            │

```

## 3. Phần 1 — Knowledge Base

### 3.1 Schema

```python
@dataclass
class ICD10Entry:
    code: str                           # Mã ICD-10 (VD: K21.9)
    name_vi: Optional[str] = None     # Tên tiếng Việt
    name_en: Optional[str] = None      # Tên tiếng Anh
    synonyms: list[str] = []           # Từ đồng nghĩa
    aliases: list[str] = []            # Alias / tên gọi khác
    description: Optional[str] = None   # Mô tả
    parent_code: Optional[str] = None  # Mã cha
    chapter: Optional[str] = None      # Chương ICD
    include_terms: list[str] = []     # Terms bao gồm
    exclude_terms: list[str] = []     # Terms loại trừ
    normalized_text: Optional[str] = None
```

## 4. Phần 2 — Normalization

### 4.1 Các bước normalization

| Bước | Mô tả | Ví dụ |
|------|--------|-------|
| Lowercase | Chuyển về chữ thường | `Viêm Phổi` → `viêm phổi` |
| Whitespace | Collapse spaces, tabs, newlines | `viêm  phổi` → `viêm phổi` |
| Dash | Thay dash bằng space | `viêm-phổi` → `viêm phổi` |
| Tones | Optional: bỏ dấu tiếng Việt | `viêm phổi` → `viem phoi` |
| Abbreviation | Expand abbreviations | `COPD` → `bệnh phổi tắc nghẽn mạn tính` |
| Synonym | Canonical form | `sốt mùa` → `viêm mũi dị ứng` |

### 4.2 TextNormalizer class

```python
from src.linking.icd.preprocess import TextNormalizer

norm = TextNormalizer(
    lowercase=True,
    remove_tones=False,     # Giữ dấu tiếng Việt
    normalize_whitespace=True,
    normalize_dashes=True,
    expand_abbreviations=True,
    expand_synonyms=True,
)

norm.normalize("Viêm-Phổi cộng  đồng")
# → "viêm phổi cộng đồng"

norm.normalize_for_fuzzy("Suy Tim")
# → lowercase, no tones, expanded

norm.normalize_for_alias("viêm-phổi")
# → "viêm phổi"
```

## 5. Phần 3 — Retrieval Sources

### 5.1 Exact Match (AliasIndex)

- Case-insensitive exact lookup
- Indexes: `name_vi`, `name_en`, `synonyms`, `aliases`, `include_terms`
- Không áp dụng synonym expansion (chỉ normalized form)

```python
results = alias_index.lookup_exact("suy tim")
# → ["I50.9"]

results = alias_index.lookup_normalized("SUY TIM")
# → ["I50.9"]
```

### 5.2 Alias Match

- Normalized exact match với alias + synonym + include_term
- Score: 0.9

### 5.3 RapidFuzz Fuzzy Matching

- So sánh mention với tất cả searchable texts
- Scorer: `fuzz.WRatio` (weighted Levenshtein)
- Score cutoff: 50
- Tự động aggregate: code với nhiều matched texts → max score

```python
results = fuzzy.retrieve("viêm phổi", top_k=10)
# → [("J18.9", 95.2), ("J20.9", 72.1), ...]
```

### 5.4 BM25 Retrieval

- Okapi BM25 (k1=1.5, b=0.75)
- Indexes: tất cả searchable texts tokenized by whitespace
- Top-30 retrieved

```python
results = bm25.retrieve("suy tim", top_k=10)
# → [("I50.9", 12.3), ("I21.9", 8.1), ...]
```

### 5.5 Dense Embedding

- Model: `intfloat/multilingual-e5-small`
- Corpus: tất cả searchable texts concatenated per entry
- Cache embeddings to `.cache/icd_dense/`
- Cosine similarity retrieval

```python
dense = DenseRetriever(
    model_name="intfloat/multilingual-e5-small",
    cache_dir=".cache/icd_dense",
)
dense.build(entries)
results = dense.retrieve("viêm phổi", top_k=10)
```

## 6. Phần 4 — Merge Strategy

### 6.1 Reciprocal Rank Fusion (RRF)

**Default** — không cần tham số.

```
score_RRF(code) = Σ  (1 / (k + rank_s)) × weight_s × (0.5 + 0.5 × norm_score)
```

- k = 60 (RRF smoothing parameter)
- norm_score: normalized về [0, 1] cho từng source

### 6.2 Weighted Normalized Score

```python
cfg = MergeConfig(method="weighted", weights={
    "exact": 1.0,
    "normalized": 0.95,
    "alias": 0.9,
    "fuzzy": 0.8,
    "bm25": 0.7,
    "dense": 0.75,
})
```

### 6.3 Configurable top-k

```python
retriever = HybridRetriever(entries=entries, top_k=20)
results = retriever.retrieve(query, mention="suy tim", top_k=5)
```

## 8. Phần 6 — Error Analysis

### 8.1 Error Classification

| Error Type | Mô tả |
|------------|--------|
| `exact_alias_missing` | Alias đúng nhưng không trong index |
| `synonym_missing` | Synonym đúng nhưng không được index |
| `translation_issue` | Mention EN nhưng KB VI (hoặc ngược lại) |
| `abbreviation_issue` | Abbreviation chưa được expand |
| `wrong_granularity` | Parent/child code mismatch |
| `dense_mismatch` | Semantic mismatch — embedding không retrieve được |
| `lexical_mismatch` | Tổng hợp — mention quá khác lexical với KB |

### 8.2 Error Output

`outputs/retrieval_errors.csv`:

```
sample_id,mention,query,gold_code,retrieved_codes,error_type,coverage
```

## 9. API và cách sử dụng

### 9.1 Basic Usage

```python
from src.linking.icd.schema import get_knowledge_base
from src.linking.icd.hybrid_retriever import HybridRetriever, MergeConfig

# Build
entries = get_knowledge_base()
cfg = MergeConfig(method="rrf", rrf_k=60)
retriever = HybridRetriever(entries=entries, merge_config=cfg, top_k=20)

# Retrieve
results = retriever.retrieve(
    query="Bệnh nhân bị suy tim",
    mention="suy tim",
    top_k=10,
)

for r in results:
    print(f"{r.code}: {r.score:.4f} from {r.sources}")
```

### 9.2 Output Format

```python
{
    "code": "I50.9",
    "score": 0.8234,
    "sources": ["exact", "fuzzy", "bm25", "dense"]
}
```

### 9.3 Individual Retrievers

```python
from src.linking.icd.alias_index import AliasIndex

from src.linking.icd.bm25_retriever import BM25Retriever

# Use individually
```
