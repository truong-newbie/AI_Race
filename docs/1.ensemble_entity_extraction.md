# Ensemble Entity Extraction System

## Overview

The ensemble entity extraction system integrates multiple information sources (regex, dictionary, and NER model) into a unified entity extraction pipeline for Vietnamese medical text.

## Architecture

```
Input Text
    │
    ├── Regex Extractor (Lab tests, lab results)
    ├── Dictionary Extractor (Drugs, diseases)
    ├── NER Model Extractor (XLM-RoBERTa fine-tuned)
    │
    ▼
Raw Entities List
    │
    ├── Group by span
    ├── Resolve type conflicts
    ├── Handle overlaps
    │
    ▼
Resolved Entities (with confidence scores)
```

## Entity Schema

Each extracted entity follows this schema:

```json
{
    "text": "Paracetamol",
    "start": 10,
    "end": 21,
    "type": "THUỐC",
    "confidence": 0.92,
    "source": "regex,dict,ner_model",
    "source_scores": {
        "regex": 0.85,
        "dict": 0.88,
        "ner_model": 0.90
    }
}
```

## Entity Types

| Type | Description | Example |
|------|-------------|---------|
| `CHẨN_ĐOÁN` | Diagnosis | Viêm phổi, tiểu đường |
| `TRIỆU_CHỨNG` | Symptom | Ho, sốt, đau đầu |
| `THUỐC` | Medication | Paracetamol, Amoxicillin |
| `TÊN_XÉT_NGHIỆM` | Lab test name | Glucose, Cholesterol |
| `KẾT_QUẢ_XÉT_NGHIỆM` | Lab result value | 126 mg/dL, 5.2 mmol/L |

## Source Types

| Source | Description | Base Confidence |
|--------|-------------|-----------------|
| `regex` | Pattern-based extraction | 0.80 |
| `dict` | Dictionary/knowledge base lookup | 0.85 |
| `ner_model` | XLM-RoBERTa fine-tuned model | 0.75 (configurable) |
| `merged` | Entity from multiple sources | Varies |

## Merge Rules

The resolver implements 6 rules for handling entity conflicts:

### Rule 1: Exact Duplicate Merge
Same span + same type → Merge with agreement bonus

```
Entity A: "viêm phổi" [15:26] CHẨN_ĐOÁN conf=0.9 source=regex
Entity B: "viêm phổi" [15:26] CHẨN_ĐOÁN conf=0.85 source=ner
─────────────────────────────────────────────────────────────
Merged:  "viêm phổi" [15:26] CHẨN_ĐOÁN conf=0.95 source="ner,regex"
```

### Rule 2: Same Span, Different Types
Same span + different types → Prefer higher confidence + section context

```
Entity A: "ho khan" [36:44] TRIỆU_CHỨNG conf=0.75
Entity B: "ho khan" [36:44] CHẨN_ĐOÁN conf=0.80
─────────────────────────────────────────────────────────────
Resolved: "ho khan" [36:44] CHẨN_ĐOÁN conf=0.80
          (higher confidence wins)
```

### Rule 3: Overlap Resolution
Overlapping spans → Prefer higher confidence

### Rule 4: Drug Preference
For `THUỐC` entities, prefer spans with dosage information:

```
Entity A: "Paracetamol" [0:12] THUỐC conf=0.9
Entity B: "Paracetamol 500mg" [0:18] THUỐC conf=0.85
─────────────────────────────────────────────────────────────
Resolved: "Paracetamol" [0:12] THUỐC conf=0.9
          (higher confidence wins when similar)
```

### Rule 5: Lab Test/Result Never Merge
`TÊN_XÉT_NGHIỆM` and `KẾT_QUẢ_XÉT_NGHIỆM` must never be merged even with identical spans.

```
Entity A: "Glucose" [0:7] TÊN_XÉT_NGHIỆM conf=0.9
Entity B: "Glucose" [0:7] KẾT_QUẢ_XÉT_NGHIỆM conf=0.8
─────────────────────────────────────────────────────────────
Resolved: Both entities kept separate
          Conflict logged: type_pair_conflict
```

### Rule 6: Section-Based Confidence
`CHẨN_ĐOÁN` and `TRIỆU_CHỨNG` confidence adjusted by section context:

```python
section_weights = {
    "chẩn đoán": {"CHẨN_ĐOÁN": 1.2, "TRIỆU_CHỨNG": 0.8},
    "khám": {"CHẨN_ĐOÁN": 1.1, "TRIỆU_CHỨNG": 1.1},
    "điều trị": {"THUỐC": 1.2, "TRIỆU_CHỨNG": 0.9},
}
```

## Confidence Configuration

```python
@dataclass
class ConfidenceConfig:
    ner_threshold: float = 0.5           # Minimum NER confidence
    dictionary_confidence: float = 0.85   # Dictionary base confidence
    regex_confidence: float = 0.80        # Regex base confidence
    agreement_bonus: float = 0.10         # Bonus for multiple sources
    section_weights: dict = {...}         # Section-based weights
    never_merge_pairs: tuple = (         # Forbidden merge pairs
        ("TÊN_XÉT_NGHIỆM", "KẾT_QUẢ_XÉT_NGHIỆM"),
    )
```

## Conflict Logging

The system tracks three types of conflicts:

| Conflict Type | Description | Resolution |
|---------------|-------------|------------|
| `same_span_diff_type` | Same span, different entity types | Highest confidence wins |
| `overlap_same_type` | Overlapping spans, same type | Highest confidence wins |
| `type_pair_conflict` | Forbidden pair detected | Both entities kept |

### Usage

```python
from src.entity.conflict_logger import ConflictLogger

logger = ConflictLogger(enable_logging=True)
report = logger.get_report()

print(f"Total conflicts: {report.total_conflicts}")
for conflict in report.conflicts:
    print(f"  - {conflict.type}: {conflict.text}")
```

## Ensemble API

### Basic Usage

```python
from src.entity.ensemble import EntityEnsemble
from src.entity.confidence import ConfidenceConfig

config = ConfidenceConfig()
ensemble = EntityEnsemble(config=config)

# Set extractors
ensemble.set_extractors(
    regex_extractor=my_regex_extractor,
    dictionary_extractor=my_dict_extractor,
    ner_extractor=my_ner_model,
)

# Extract entities
entities = ensemble.extract(text, section="chẩn đoán")
```

### Using EntityResolver Directly

```python
from src.entity.resolver import EntityResolver
from src.entity.conflict_logger import ConflictLogger
from src.entity.confidence import ConfidenceConfig

config = ConfidenceConfig()
logger = ConflictLogger(enable_logging=True)
resolver = EntityResolver(config=config, conflict_logger=logger)

# Collect entities from all sources
entities = []
entities.extend(regex_extractor(text))
entities.extend(dictionary_extractor(text))
entities.extend(ner_model.predict(text))

# Resolve conflicts
resolved = resolver.resolve_entities(entities, text, section="chẩn đoán")
```

## Best Practices

1. **Always use original text** for span extraction to ensure accurate text values
2. **Configure thresholds** based on your use case (higher = more precision, lower = more recall)
3. **Log conflicts** during development to understand edge cases
4. **Section context** improves disambiguation for CHẨN_ĐOÁN/TRIỆU_CHỨNG
5. **Never merge lab entities** - they have different semantic meanings

## Performance Considerations

- Entity resolution is O(n²) for overlap detection
- Grouping by span first reduces comparisons
- For large texts, consider batch processing
- Confidence scoring adds minimal overhead
